# Research Intake

Research is useful only when it closes a repo-proven capability gap.

## When To Research

Research when:

- a required capability is missing
- an API/library/tool may have changed
- current docs or prices matter
- open-source alternatives should be compared
- a paid tool might be avoidable
- security, licensing, or data rights matter

## Research Record

For every new tool or source, record:

```yaml
name:
url:
license:
category:
why_needed:
replaces:
install_command:
proof_command:
risks:
decision: adopt | watch | reject
date_checked:
```

## Decision Rules

- Prefer official docs and primary repositories.
- Prefer boring tools with tests, active maintenance, and clear licenses.
- Do not add a framework when a script or skill is enough.
- Do not add paid APIs until existing sources fail with exact blockers.
- Do not confuse "can automate" with "safe to automate."
