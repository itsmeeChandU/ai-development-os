# Governance

AI Development OS is founder-led.

## Maintainer Authority

Sai Chandra is the project owner and final decision maker for:

- roadmap
- release direction
- licensing posture
- commercial/private/association exceptions
- trademark and attribution policy
- accepted contribution terms
- official project claims
- security disclosure handling

## Project Intent

The project exists to help AI agents produce real, verified software from:

- startup ideas
- existing codebases
- current-state prompts
- messy product handoffs
- source/data/tool audits

The project should not drift into a generic prompt library or an unverified
agent benchmark collection.

## Change Standard

Major changes should preserve:

- state reconstruction before broad edits
- system review graph before complex implementation
- file/module ownership for agents
- generated artifacts as current truth
- proof gates and blocker rows
- attribution and copyleft requirements
- no implied responsibility for derivative products

## Commercial And Association Decisions

Only the project owner can approve:

- private/proprietary licenses
- attribution changes
- official partner/association claims
- certification claims
- trademark use beyond accurate attribution
- patent-style commercialization arrangements
