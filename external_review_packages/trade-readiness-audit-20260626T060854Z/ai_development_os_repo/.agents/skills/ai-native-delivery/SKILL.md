# AI Native Delivery Skill

1. Read the startup brief or ask for one if the idea is too vague.
2. Build a system review graph: code, data, flow, proof, task, and resources.
3. Create module contracts before splitting agents.
4. Spawn or assign agents only with allowed files, forbidden files, tests, and handoff format.
5. Require every lane to produce code/data/tests/artifacts or a concrete blocker.
6. Use research only when the system review graph shows a missing capability.
7. Keep one coordinator responsible for integration and final verification.
8. Do not mark done without generated evidence and focused checks.
