# Licensing

AI Development OS uses a strong copyleft posture.

## Public License

The public license is:

```text
AGPL-3.0-or-later
```

That means public users may use, modify, and share the project, but if they
distribute modified versions or make modified versions available as a network
service, they must provide the corresponding source under the same license
terms.

## Attribution

Modified versions, forks, distributions, and network services based on this
project must preserve the notice in `NOTICE.md`.

Required attribution:

```text
Built with AI Development OS by Sai Chandra.
```

## Original Copyright Holder

The original copyright holder may grant separate commercial, private, or
alternative licenses. This lets the creator use the process differently while
the public project remains strongly copyleft for everyone else.

## Commercial Or Association Exceptions

The public path is open source and copyleft. Different terms require a separate
written agreement from the copyright holder.

Examples that require a separate agreement:

- private/proprietary derivative process
- closed-source hosted product based on this process
- attribution removal or modified attribution
- official association, endorsement, certification, or partner status
- commercial integration that does not want AGPL obligations
- patent-style commercialization or exclusive field-of-use arrangement

See `docs/COMMERCIAL_AND_ASSOCIATION.md` and `TRADEMARKS.md`.

## Product Responsibility

Using this process does not make the project owner responsible for derivative
products. Product owners are responsible for their own outputs, claims, data,
deployments, compliance, transactions, and harms. See `DISCLAIMER.md`.

## Practical Agent Rule

When an agent creates a derivative public process from this repo, it should:

1. keep `LICENSE`
2. keep `NOTICE.md`
3. use AGPL-compatible licensing for derivative process code
4. preserve the attribution line in docs or an About/legal notice
5. document any third-party dependency licenses in the research/tool registry
6. stop if asked to create a proprietary exception without a written license

This file is implementation guidance, not legal advice.
